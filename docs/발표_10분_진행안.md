## 3. "AI 활용법" 발표 포인트 (2:15–4:15, 가장 차별화)

발표의 핵심 메시지: **"AI를 어디에·어떻게·왜 쓰는가"** — 기술 자랑이 아니라 업무 가치로.

**슬라이드 ④ RAG 파이프라인 (1분)**
- 원칙 한 줄: **"DB가 원본, Chroma는 검색 인덱스"** (단일 진실 공급원)
- `판례 JSONL → DB → 임베딩(BGE-M3) → Chroma`, 검색 = **Dense + BM25 → RRF 융합**
- 결과는 식별자+점수만, **본문은 DB에서 조립** → 근거가 항상 원본

**슬라이드 ⑤ 업무별 AI + 데이터 흐름 (1분)**
- 업무 매핑(표 한 장):
  - 의뢰인: **키워드 추출**(→변호사 검색에 재사용), **법리 설명**(근거 판례 인용)
  - 변호사: **사건 요약**, **유사판례 정리**
- 공통 원리: **요청→호출→저장**(ai_analysis_result에 사건별 이력 누적)
- "키워드 추출 → 그 키워드로 변호사 검색"의 **연결**을 데모에서 보여줄 것이라 예고

## 4. 데모에서 "무엇을" 집중할지 — 아이디어

핵심 원칙 ▶ **모든 메뉴를 클릭하지 말고, 한 사건을 한 줄기로**. 3.5분 안에 "AI 가치"와 "워크플로우"를 보여주는 4개의 장면에만 집중.

**집중 장면 (Wow 포인트)**
1. **키워드 추출 → 변호사 검색 연결** ★최우선
   - 사건 사실관계 → AI 키워드 추출(즉시) → 그 키워드로 변호사 검색 매칭.
   - "AI 결과가 다음 업무의 입력이 된다"는 가장 직관적 가치.
2. **RAG 법리/유사판례의 "근거"** ★
   - 결과에 **인용 판례(case_id)** 가 함께 표시 → "환각이 아니라 DB 판례 기반" 강조.
3. **역할별 권한** (10초 컷)
   - 의뢰인으로 변호사 전용 URL 직접 접근 → **403** → "서버에서 강제" 한 컷.
4. **상태 변화 실시간**
   - 수임 요청 → 대표 조건 전달/확정 → 사건 상태가 **RETAINED**로 변하는 장면(상태머신 가시화).

**일부러 피할 것**
- 빈 폼 타이핑(미리 채워두기), 모든 CRUD 클릭, **느린 LLM 대기**(요약·법리는 수십초).

## 6. 데모 안정화 팁 (리허설 필수)
- **사전 시드 활용**: 회원(pw1234)·판례·담당사건이 자동 생성 → 데이터 만드는 시간 0.
- **느린 AI 대비**: 요약·법리는 **데모 직전 1회 미리 실행**(결과 캐시) 또는 가벼운 모델(`qwen2.5:7b`). 실패 시 **백업 스크린샷 슬라이드** 준비.
- **graceful degradation을 역이용**: AI가 비어도 "장애 시 빈 결과로 안전하게 떨어지는 설계"라고 설명 가능.
- 브라우저 **탭 2개**(의뢰인/변호사) 미리 로그인 → 역할 전환을 끊김 없이.
- 글씨 크게(확대), 마우스 동선 최소화, 클릭 전에 "지금 무엇을 보여줄지" 한마디.
