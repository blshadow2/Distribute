## 슬라이드 7 — 수임 라이프사이클 (상태머신) ★
- 사건등록 → 변호사검색 → 상담(신청·승인) → 수임요청
- **수임 상태머신**:
  `REQUEST_SENT → CONDITION_SENT ⇄ CONDITION_ADJUSTMENT_REQUESTED → CONDITION_ACCEPTED → RETAINED`
  (거절: 의뢰인/대표 양측 → REJECTED)
- 전이 검증은 **서비스 계층**에서(잘못된 전이 차단)
- 수임 확정 → 사건 RETAINED + 담당변호사 지정
- [시각자료] 상태 전이 다이어그램

## 슬라이드 8 — 판례 RAG: 데이터 파이프라인 ★
- 원칙: **DB가 원본(precedent), Chroma는 검색 인덱스**(단일 진실 공급원)
- `privacy_cases.jsonl → (JPA)precedent → build_index → Chroma`
- 청킹: 판례를 **chunk_type**(판시사항·판결요지·참조조문·전문) 단위로 분할·임베딩
- 검색 결과는 **external_case_id + 점수만** 반환 → 본문은 Java가 DB 조회
- [시각자료] 파이프라인 흐름도(원본↔인덱스 분리 강조)

## 슬라이드 9 — RAG 검색 알고리즘 상세 ★ (신규)
- **Dense**: BGE-M3 정규화 임베딩 → Chroma cosine, top-K
- **Sparse**: BM25Okapi(한글 토크나이저: 한/영/숫자 ≥2자), top-K
- **융합(RRF)**: `score(d)=Σ 1/(K+rank_i(d))`, K=60 — dense·sparse 순위 결합
- **chunk_type 가중치**: 판시사항 1.15 / 판결요지 1.10 / 참조조문 1.05 / 전문 1.00
- **case_id 그룹핑**(청크 점수 합산) → **min-max 정규화(0~1)** → Java threshold 호환
- [시각자료] dense+sparse→RRF→그룹핑 단계도

## 슬라이드 10 — AI LLM 기능 + 영속화 ★
- 요약 / 키워드 / 법리(RAG 결합) / 유사판례
- Ollama `/api/chat`, **format="json"** 으로 구조화 강제, temp 0.2, num_ctx
- **파싱 견고화**: 코드펜스 제거 + 키 변형 허용 + **산문 폴백**(작은 모델 대응)
- **요청–호출–저장**: `ai_analysis_request`(FK 부모) → 기능 실행 → `ai_analysis_result` 저장(case_id 연결)
- [시각자료] 요청-호출-저장 시퀀스 + JSON/폴백 분기

## 슬라이드 11 — Java ↔ Python 통신 계약 ★ (신규)
- 클라이언트: JDK 내장 `HttpClient` + Jackson, **HTTP/1.1 고정**
- 계약(REST): `/search/similar-precedents`, `/answer`, `/llm/*`, `/rag/analyze-rules`
- **경량 응답 설계**: 검색은 식별자+점수만 → 본문 중복 제거(단일 원본)
- **graceful degradation**: RAG/LLM 장애 시 빈 결과·폴백 → 기능 전체 비중단, 타임아웃(LLM 120s)
- [시각자료] 요청/응답 JSON 예시

## 슬라이드 13 — 인증 · 역할 권한 (서버 강제) ★
- 경량 **세션 인증**(HttpSession) + `LoginInterceptor.preHandle`

  (페이지: /access-denied, `/api/**`: 403 JSON)
- 화면 메뉴 숨김(UX) + 서버 차단(보안) = **2중 방어**
- 비밀번호 **BCrypt** 해시
- [시각자료] 요청→인터셉터 분기 흐름

## 슬라이드 15 — 분산 시스템 설계 관점 ★ (신규)
- **프로세스 분리**: 웹/AI/DB/LLM 독립 → 독립 배포·확장, **장애 격리**
- **공유 상태 일관성**: 단일 DB를 Java(JPA)·Python(pymysql)이 동일하게 참조(`LAWSYSTEM_DB_URL` 일원화)
- **멱등성**: 시드(@Order + existsBy*), 판례 **결정적 PK**(PREC-…) upsert → 재시작/재실행 안전
- **경계 명확화**: RAG=검색만, 본문은 DB 원본 → 중복·불일치 방지
- **부분 실패 허용**: graceful degradation(빈 결과/폴백)
- [노트] "분산프로그래밍" 핵심 어필 슬라이드

## 슬라이드 16 — 핵심 설계 결정 (이유 중심)
- Java/Python **HTTP 분리** ← 임베딩·LLM은 Python 전용, 모델 상주 필요
- **단일 DB + JPA 스키마 권한** ← schema.sql(JDBC)과 상속 구조 비호환 해소
- **HTTP/1.1 고정** ← uvicorn HTTP/2 미지원으로 POST 본문 유실
- **LLM JSON+산문 폴백** ← 모델 품질 편차(gemma 등) 대응
- [노트] 결정→근거 1:1

## 슬라이드 17 — 트러블슈팅 / 배운 점
- **HTTP/2 422**(본문 유실) → HttpClient HTTP/1.1 고정으로 해결
- 스키마 이중화(JDBC vs JPA) → 단일 DB(JPA) 일원화
- 런타임: Python 3.8→3.13, RTX50 CUDA cu128/CPU 폴백
- gemma JSON 약점 → 산문 폴백
- [노트] "통합/분산에서 실제 겪은 문제"로 설득력
